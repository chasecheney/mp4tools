# MP4tools+ — Setup Guide

How to turn the source in this repo into a running macOS app. Estimated time: ~15 minutes.

## Prerequisites

- **macOS 14 (Sonoma) or later**
- **Xcode 15 or later** — from the Mac App Store or [developer.apple.com](https://developer.apple.com/xcode/)
- **ffmpeg + ffprobe** — install via Homebrew:

  ```sh
  brew install ffmpeg
  ```

  This puts `ffmpeg` and `ffprobe` in `/opt/homebrew/bin` (Apple Silicon) or `/usr/local/bin` (Intel). The app's `BinaryLocator` discovers them automatically. (For a self-contained, distributable build, see [Bundling ffmpeg](#bundling-ffmpeg-optional) below.)

## 1. Create the Xcode project

The Swift sources are drop-in, but the `.xcodeproj` is best generated by Xcode itself.

1. Open Xcode → **File ▸ New ▸ Project…**
2. Choose **macOS ▸ App**, then **Next**.
3. Fill in:
   - **Product Name:** `MP4toolsPlus`
   - **Interface:** SwiftUI
   - **Language:** Swift
   - **Bundle Identifier:** `com.cheney.MP4toolsPlus` (match `Info.plist`)
4. Save it **inside this repo folder** (`~/claude/mp4tools`). Let it create the `MP4toolsPlus.xcodeproj`.
5. Xcode generates a starter `MP4toolsPlusApp.swift` and `ContentView.swift` — **delete those two** from the project (move to Trash); the real versions live in `MP4toolsPlus/`.

## 2. Add the source files

1. In the Xcode Project navigator, right-click the project → **Add Files to "MP4toolsPlus"…**
2. Select the `MP4toolsPlus/` folder from this repo.
3. Check **"Create groups"** (not folder references) and make sure the app target is ticked.
4. You should now see the `App`, `Models`, `ViewModels`, `Views`, `Services`, and `Resources` groups.

## 3. Wire up Info.plist and entitlements

> **Heads-up for modern Xcode (13 through 26.x):** new SwiftUI apps no longer create a separate `Info.plist` file — Xcode synthesizes one at build time, so the **Info.plist File** build setting is empty by default and there's nothing to point at until you turn generation off. Use **Option A** below to use the `Info.plist` in this repo.

### Option A — use the repo's Info.plist (recommended)

1. Select the project → select the **MP4toolsPlus target** → **Build Settings** tab.
2. Set the filter to **All** (not Basic) and type `info.plist` in the search box.
3. Under *Packaging*, set:
   - **Generate Info.plist File** → **No**
   - **Info.plist File** → `MP4toolsPlus/Resources/Info.plist`  (this is the `INFOPLIST_FILE` setting; it becomes editable once "Generate" is No)

### Option B — keep the auto-generated plist

Leave **Generate Info.plist File** = Yes and add custom keys yourself:

- Select the target → **Info** tab → under *Custom macOS Application Target Properties*, click **+** to add keys. This is where `CFBundleDocumentTypes` (the drag-drop file types) lives. Simple scalar keys can instead be added in Build Settings as `INFOPLIST_KEY_*` entries.
- Note: `CFBundleDocumentTypes` is a nested array and tedious to enter by hand, so Option A is usually less work for this project.

### Entitlements

1. In **Build Settings**, search `entitlements` and set **Code Signing Entitlements** (`CODE_SIGN_ENTITLEMENTS`) to `MP4toolsPlus/Resources/MP4toolsPlus.entitlements`.
   - Alternatively, use **Signing & Capabilities ▸ + Capability ▸ App Sandbox**, which creates and links an entitlements file for you.
2. Under **Signing & Capabilities**, confirm **App Sandbox** is on with **User Selected File ▸ Read/Write** enabled (the entitlements file already declares this).

> Exact label wording may vary slightly in Xcode 26.4.1, but these settings live under **Build Settings → search "info.plist" / "entitlements" → All filter**.

## 4. Build and run

Press **⌘R**. The app launches to the drop zone. Drag in an MKV/MP4/WEBM/AVI file and it should probe instantly via ffprobe, listing the tracks.

If you see *"Could not find ffmpeg"*, ffmpeg isn't on a discovered path — confirm `which ffmpeg` returns a path under `/opt/homebrew/bin` or `/usr/local/bin`, or bundle the binaries (below).

## Bundling ffmpeg (optional)

For an app you can hand to someone without Homebrew:

1. Download static `ffmpeg` and `ffprobe` builds (e.g. from [evermeet.cx/ffmpeg](https://evermeet.cx/ffmpeg/) for macOS).
2. Drag both binaries into the **Resources** group; ensure they're in **Copy Bundle Resources** for the target.
3. `BinaryLocator` checks the bundle first, so they'll be used automatically.
4. Note: a sandboxed app spawning bundled binaries may need a hardened-runtime exception or a privileged helper for full Mac App Store compliance. For direct distribution (notarized DMG), the current entitlements are sufficient.

## Sandbox note

The app runs in the macOS App Sandbox and only touches files the user explicitly drags in or opens. To process files unattended (e.g. a watch-folder feature later), you'd persist security-scoped bookmarks — the `files.bookmarks.app-scope` entitlement is already declared for that.

## Troubleshooting

| Symptom | Fix |
| --- | --- |
| "Could not find ffmpeg/ffprobe" | `brew install ffmpeg`, or bundle static binaries |
| ffprobe reads file but no tracks show | Check the file isn't DRM-protected; try another container |
| Conversion fails immediately | Open the job's error text in the tray; usually a codec that can't be copied into MP4 — switch the preset to re-encode |
| Subtitles don't appear | MP4 soft subs require `mov_text`; the muxer handles this, but source bitmap subs (PGS/VOBSUB) must be **burned in**, not muxed |
| Build errors on `onChange(of:)` / two-param API | Confirm the deployment target is macOS 14+ |
