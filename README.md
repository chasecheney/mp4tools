# MP4tools+

A native macOS app (Swift + SwiftUI) for fast, mostly-lossless video processing built on **ffmpeg** / **ffprobe**. Drop a video in, pick what you want, and let it run — most jobs stream-copy tracks so they finish in minutes with no quality loss.

## Features (copied from EmmGunn Software MP4tools)

- **Drag-and-drop intake** for MKV, MP4, WEBM, OGM, AVI and more, probed with ffprobe.
- **Convert to MP4** — copy video/audio tracks (no re-encode) whenever possible.
- **Surround sound** conversion to 5.1 AC-3 or 5.1 AAC.
- **Track selection** for video / audio / subtitle streams, with an AVKit preview to verify.
- **Presets** for one-step encoding (iPhone, Apple TV, remux) plus user-defined automated presets.
- **Subtitles** — burn-in (hard) or mux (soft), from internal tracks or external `.srt` / `.ass` files.
- **Split** by max segment size or by start/end time.
- **Join** multiple files (lossless concat).
- **Extract** selected tracks as individual files.
- **Adjust pixel aspect ratio** without re-encoding.

## Architecture (MVVM)

```
MP4toolsPlus/
├── App/
│   └── MP4toolsPlusApp.swift     // @main, WindowGroup + Settings scene, menu commands
├── Models/
│   ├── MediaModels.swift         // MediaFile, MediaTrack, TrackKind
│   ├── Preset.swift              // Preset + StreamMode/AudioTarget/SubtitleMode enums
│   └── Job.swift                 // Operation, JobStatus, observable Job
├── ViewModels/                   // @MainActor, ObservableObject
│   ├── LibraryViewModel.swift    // intake, probing, track edits, auto-selection
│   ├── PresetStore.swift         // presets persisted to UserDefaults
│   └── JobQueueViewModel.swift   // sequential job queue + progress + alerts
├── Services/                     // no UIKit/SwiftUI; testable engine
│   ├── ProcessRunner.swift       // async/await Process wrapper, streams stderr
│   ├── BinaryLocator.swift       // finds bundled or Homebrew ffmpeg/ffprobe
│   ├── MediaInspector.swift      // ffprobe JSON → MediaFile
│   ├── FFmpegCommandBuilder.swift// pure functions: Operation → ffmpeg args
│   ├── FFmpegService.swift       // executes operations, parses progress
│   └── OutputNaming.swift        // collision-free output URLs
├── Views/                        // SwiftUI, native macOS idioms
│   ├── ContentView.swift         // NavigationSplitView + drop handling
│   ├── LibrarySidebar.swift      // sidebar list
│   ├── DropZoneView.swift        // empty-state drop target
│   ├── DetailView.swift          // preview, presets, actions
│   ├── TrackTableView.swift      // per-track checkboxes
│   ├── OperationSheet.swift      // split / join / extract / PAR
│   ├── JobTrayView.swift         // live progress tray
│   └── SettingsView.swift        // ⌘, Settings + preset editor
└── Resources/
    ├── Info.plist
    └── MP4toolsPlus.entitlements
```

### Key design points

- **Concurrency.** All external-tool work happens in actors (`ProcessRunner`, `MediaInspector`, `FFmpegService`) using `async/await`. View models are `@MainActor` so UI state mutations are safe.
- **Progress.** `FFmpegService` parses `time=` from ffmpeg's stderr and divides by the probed duration to emit a 0–1 fraction.
- **Testability.** `FFmpegCommandBuilder` is pure (no I/O), so the argument vectors for every operation can be unit-tested directly.
- **Lossless by default.** Conversions and split/join/extract use `-c copy` unless a preset asks to re-encode or subtitles need burning in.

## Requirements & setup

1. **Xcode 15+**, macOS 14+ target.
2. Create a new macOS App in Xcode (SwiftUI lifecycle), then add the `MP4toolsPlus/` group, set the entitlements file, and point Info.plist settings as shown.
3. **ffmpeg / ffprobe.** Either:
   - `brew install ffmpeg` (the app discovers `/opt/homebrew/bin` or `/usr/local/bin`), or
   - drop static `ffmpeg` and `ffprobe` binaries into the app bundle's Resources for a self-contained build (preferred for distribution).

> Note: `.xcodeproj` is best generated by Xcode. The sources here are drop-in; create the project shell in Xcode and add this folder.
